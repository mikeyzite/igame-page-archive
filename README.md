# igame-page-archive

本仓库用于归档和发布多个独立的 HTML 页面。这些页面是独立于任何特定平台或域名的静态资源，适用于存档、展示或离线浏览。

## 项目简介

igame-page-archive 是一个以 HTML 页面为主要内容的归档仓库。每个页面都独立封装，不依赖外部框架或数据库，可直接在浏览器中运行。本项目主要面向需要保存和分享静态页面内容的场景。

## 目录说明

```
├── pages/          # 存放所有归档的 HTML 页面
│   ├── example/    # 示例子目录（如无实际内容可忽略）
│   └── ...
├── assets/         # 页面所需的公共资源（样式、脚本、图片等）
├── index.html      # 仓库首页（可选）
└── README.md       # 本文件
```

- `pages/`：每个子目录对应一个独立的归档页面，目录名可反映页面内容或编号。
- `assets/`：存放多个页面共用的资源文件，以减少重复。
- 页面文件命名建议使用英文或数字，避免特殊字符。

## 页面归档说明

归档的页面均满足以下基本要求：

- 所有 HTML 文件均为独立、自包含的静态页面。
- 页面资源（CSS、JavaScript、图片等）优先内联或置于 `assets/` 目录下。
- 页面不包含跟踪代码、外部 API 调用或动态数据加载。
- 每个页面可在本地直接打开运行，无需网络环境。

若需新增归档页面，请将文件放入 `pages/` 下对应子目录，并确保引用路径正确。

## 维护说明

本仓库由维护者定期整理和更新。欢迎提交 Pull Request 添加新页面或改进现有页面，但请注意：

- 提交内容需为完全静态的 HTML 文件。
- 不得包含任何形式的推广、跟踪或外部请求。
- 请确保页面无恶意代码或敏感内容。

如发现页面存在问题（如链接失效、样式错乱），欢迎通过 Issue 反馈。
